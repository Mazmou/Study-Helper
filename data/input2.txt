The Transmission Control Protocol (TCP) is a connection-oriented transport protocol designed to provide reliable, ordered, and error-checked delivery of data between applications running on hosts communicating via an IP network. TCP establishes a connection using a three-way handshake mechanism, which consists of SYN, SYN-ACK, and ACK segments exchanged between client and server before data transmission begins.

The Internet Protocol (IP) is a network-layer protocol responsible for addressing and routing packets across network boundaries. Each device connected to a network is assigned a unique IP address, which enables packets to be forwarded correctly from source to destination. IP is considered a connectionless protocol because it does not guarantee delivery, ordering, or duplicate protection.

The User Datagram Protocol (UDP) is a lightweight transport protocol that provides minimal services compared to TCP. UDP does not establish a connection before transmitting data and does not provide reliability, flow control, or congestion control. Because of this, UDP is often used in time-sensitive applications such as real-time video streaming, online gaming, and Voice over IP (VoIP).

Congestion control in TCP is implemented through algorithms such as Slow Start, Congestion Avoidance, Fast Retransmit, and Fast Recovery. During the Slow Start phase, the congestion window increases exponentially until packet loss is detected or a threshold is reached. After that, the congestion window grows linearly to avoid overwhelming the network.

The TCP/IP model divides networking responsibilities into four layers: the Application layer, the Transport layer, the Internet layer, and the Network Access layer. The Application layer provides protocols for specific data communications services, such as HTTP for web browsing and SMTP for email transmission. The Transport layer provides end-to-end communication services for applications. The Internet layer handles logical addressing and routing. The Network Access layer is responsible for physical transmission of data over hardware.

Hypertext Transfer Protocol (HTTP) is an application-layer protocol used for transferring hypermedia documents such as HTML. HTTP is stateless, meaning each request from a client to a server is treated as an independent transaction. Secure HTTP (HTTPS) adds encryption using Transport Layer Security (TLS) to protect data integrity and confidentiality.

The Domain Name System (DNS) translates human-readable domain names into IP addresses. DNS operates using a hierarchical and distributed database structure. When a user enters a domain name in a browser, a DNS resolver queries multiple servers until the corresponding IP address is found.

Flow control ensures that a sender does not overwhelm a receiver with too much data at once. TCP uses a sliding window mechanism to manage flow control. The receiver advertises a window size that determines how much data the sender may transmit before receiving an acknowledgment.

Checksum mechanisms are used to detect errors in transmitted data. TCP includes a checksum field in its header to verify the integrity of both header and payload data. If the checksum does not match, the segment is discarded.

In contrast to TCP, UDP provides no guarantee of delivery, ordering, or duplicate protection. However, its simplicity reduces latency and overhead, making it suitable for applications that prioritize speed over reliability.
